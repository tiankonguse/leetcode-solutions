## CSP-J/S 常见暴零原因  



## 爆内存  


问题：数组开的比较大，超过 512M，导致爆内存。  



case: HA-J00423 预测 400 分，实际 300 分，polygon 暴零，错失第一名，可惜了。  


```cpp
#define int long long
const int N = 5e3 + 5, mod = 998244353;
int n, a[N], maxn, dp[N][N], sum[N][N], sum2[N][N];
```


## 漏传代码  


问题：提交时漏传代码文件，导致暴零。  


case: HA-J00664, 漏传代码，得分 280，看他的实力，至少能得 350 分。  




## 函数返回值问题  


问题：函数定义返回值，最后没有返回值  


case HA-S00524, 预期最后1题得暴力分，总分 138。 由于函数没有返回值，只得了 96 分。  


## 数组下标越界  

问题：定义的数组大小为 N，从 1 开始遍历，遍历到 N，导致数组下标越界。  

case HA-S00072，少的 36 分。  


## 没有输出到文件  


问题：调试时临时把输出改成了屏幕输出，提交时忘记改回文件输出，导致暴零。  



## 保存时误输入字母  


问题：按快捷键 `ctrl+s` 保存时，在代码最后输入一个字母 s，导致编译错误，暴零。
